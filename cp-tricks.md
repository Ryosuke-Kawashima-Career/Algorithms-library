## Algorithms

### Search (探索)

全探索: 1．変数を減らす　２．全探索の対象を変える。
半分全列挙: 全体を二つに分けてbit全探索 -> 二分探索 することで全探索を高速化する。
再帰: 同じことの繰り返し。ベースケースに注意。
再帰: 行きがけ(親の情報を子供に)と帰りがけ(子供の情報を親に) -> 再帰呼び出しの前か後か？

### Binary Search (二分探索)

二分探索: 探索の範囲を狭める。答えを特定したとき条件を検証できるときに二分探索を使用する。
二分探索: 最大値の最小化の問題に使える。
二分探索: 答えを対象に行うことも, 条件が分岐する境界がわかる。

### DP (Dynamic Programming)

DP: 部分問題をメモ化でまとめる。
DPでは添え字に意味を持たせる ex.場合の数 = `dp[i: 1からiまで使える][j: value, state!!!(ex. bit)]`
添え字のjとして直前の状態を記録する場合がある．

DPで何も使用しないとき, `dp[i][j] = dp[i-1][j];`
DP: 添え字の入力の大きさで添え字を決める。
Dp: `dp[0]`などのindex=0の時の初期化を忘れない。
区間DP: 操作を l:-> <- :r もしくは l: <- -> : r で行うかでfor文の回し方が違う。

### Greedy (貪欲法)

greedy: 完全上位互換の状況にする。ソートと相性がいい。
greedy: 辞書順と相性がいい。

### Data structure (データ構造)

区間クエリ: prefix, segment tree(区間を変更する->Lazy), fenwick(BIT)
条件を満たす範囲: 尺取り法, ランレングス圧縮

### Flow (フロー)

マッチング問題: 最大流, DP

多次元配列のindex -> 一次元index: y * w + x
一次元 -> 多次元: i: node / w, j: node % w

dfs: 全探索 -> seenを復元する, 一例のみの場合 -> 復元の必要なし

hash: indexの拡張版みたい(Hashset, Hashmap)
hash: ある値が存在するかをO(1)で行える。
hash: あらかじめ範囲のhash値を計算し、文字列を比較する。
二分木: 値の変更、消去、探索をO(logn)で行う。
ヒープ: 最大最小の値の計算を高速に行う。
